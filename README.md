# ShieldProxy — LLM Prompt-Injection Firewall 🛡️

> **Tagline:** Every Indian startup is building AI chatbots. None of them are secure. ShieldProxy is the missing security layer.

ShieldProxy is a highly-performant, production-grade reverse proxy API that sits seamlessly between your application's users and backend upstream LLMs (GPT-4, Gemini, Claude). Before a prompt reaches the LLM, it evaluates and safely processes the payload across a robust **4-Phase Security Pipeline** to ensure total enterprise safety against prompt injection, jailbreaks, and PII leakage.

---

## 🌟 Core Architecture & The 4-Phase Pipeline

### 1. The Gateway & Rate Limiter (Golang + Redis)
Built in Go (`net/http`) utilizing a lightning-fast Lua script Atomic Token Bucket algorithm backed by Redis. Limits throughput per API key globally preventing Denial-of-Wallet (DoW) attacks prior to computation. (Fails open locally if Redis is disconnected).

### 2. The Threat Classifier (Llama 3 8B + Sub-10ms Caching)
Prompts are diverted to an isolated, locally-hosted **Llama 3** engine dynamically constrained by strict `temperature: 0.0` deterministic Few-Shot behavioral rules. It accurately classifies queries into 5 threat vectors including:
- **Jailbreaks** (DAN, Developer Overrides)
- **Prompt Leaks** (System Instruction Harvesting)
- **Social Engineering** (Hinglish/Colloquial Bypass)

⚡ **Performance Upgrade:** Includes a thread-safe `sync.Map` Semantic LRU Cache. Repeat attacks bypass the GPU entirely and return verdicts in `<5ms`. 
🚨 **Fallback Engine:** If Ollama goes offline, the proxy natively pivots to a Regex string-matching heuristic subroutine ensuring 100% fail-secure uptime.

### 3. PII Redaction Scrubber (Python / FastAPI + spaCy)
If the prompt passes the threat phase without malice, it drops into a lightweight Python microservice on port `:5001`. Here, a dual-pass algorithm completely redacts Personal Identifiable Information (PII) before the payload leaves the network.
- **Pass 1:** Pre-compiled Regex patterns (`SSN, Phone, Credit Card, Email`)
- **Pass 2:** N-Gram Contextual `spaCy` NER (`PERSON, ORG, LOC, MONEY`)

### 4. Output Filter & Real-Time Telemetry Node
All outbound responses generated by the upstream API are re-evaluated backwards through a lightning-fast **Output Filter** checking for data leakages or rule-breaks.
Finally, the asynchronous JSON **Audit Logger** captures full telemetry (`LatencyMs`, `Verdict`, `Category`, `Confidence`, `Intercepted_Prompt`) and dynamically serves it over a dedicated API Endpoint (`/v1/audit/logs`).

## 📊 The "Digital Vault" Intelligence Dashboard

We built a state-of-the-art React.js (Vite) interface styled using raw CSS ("The Digital Vault" Design System) to visualize the firewall in action.

- **Attack Simulator Engine:** An interactive lab where you can inject custom prompt injections, select pre-built exploit payloads, and route them natively through the Go backend to instantly receive evaluation stats.
- **Network Sovereignty Overview:** A real-time matrix dashboard. It natively polls the Go Server's `/v1/audit/logs` every `2000ms`, interleaving authentic logs with background noise. Incoming actual threats **visually flash neon-green and pop into the Interception Feed exactly as they occur.**

## 🚀 Native Local Deployment

### 1. Start the React Frontend Dashboard
```bash
npm install
npm run dev
```
*(Runs on exactly `localhost:5173`)*

### 2. Start the Python PII Scrubber
```bash
cd python-scrubber
pip install -r requirements.txt
python -m spacy download en_core_web_sm
python -m uvicorn main:app --port 5001
```
*(Runs on exactly `localhost:5001`)*

### 3. Start the Go Firewall Proxy
Verify Ollama is installed and running (`llama3`) locally on port `11434`.
```bash
cd go-backend
go mod tidy
go run ./cmd/proxy/main.go
```
*(Proxy listens globally on `http://localhost:8080`)*
