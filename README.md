# ShieldProxy — LLM Prompt-Injection Firewall 🛡️

> **Tagline:** Every Indian startup is building AI chatbots. None of them are secure. ShieldProxy is the missing security layer.

ShieldProxy is a highly-performant, production-grade reverse proxy API that sits seamlessly between your application's users and backend upstream LLMs (GPT-4, Gemini, Claude). Before a prompt reaches the LLM, it evaluates and safely processes the payload across a 4-Phase Security Pipeline to ensure total enterprise safety against prompt injection, jailbreaks, and PII leakage.

---

## 🌟 The 4-Phase Security Architecture

### 1. The Gateway & Rate Limiter (Golang + Redis)
Built in Go (`net/http`) utilizing a lightning-fast Lua script Atomic Token Bucket algorithm backed by Redis. Limits throughput per API key (60 req/min) globally preventing Denial-of-Wallet (DoW) attacks prior to computation.

### 2. The Threat Classifier (Llama 3 8B + Few-Shot Tuning)
Prompts are diverted to an isolated, locally-hosted **Llama 3** engine dynamically constrained by strict `temperature: 0.0` deterministic Few-Shot behavioral rules. It accurately classifies queries into 5 threat vectors including:
- Jailbreaks (DAN, Developer Overrides)
- Prompt Leaks (System Instruction Harvesting)
- Hinglish/Colloquial Social Engineering

If Ollama is overwhelmed or offline, the Go firewall instantly triggers a deterministic Regex string-matching **Fallback Subroutine** ensuring 100% uptime fail-open/closed business logic.

### 3. PII Redaction Scrubber (Python / FastAPI + spaCy)
If the prompt passes the threat phase without malice, it drops into a lightweight Python microservice. Here, a dual-pass algorithm completely redacts Personal Identifiable Information (PII) before the payload hits upstream cloud inference targets.
- **Pass 1:** Pre-compiled Regex patterns (`SSN, Phone, Credit Card, Email`)
- **Pass 2:** N-Gram Contextual `spaCy` NER (`PERSON, ORG, LOC, MONEY`)

### 4. Output Filter & Async Audit Logging
All outbound responses generated by the LLMs are re-evaluated backwards through a lightning-fast **Output Filter** checking for data leakages or model hallucinatory rule-breaks.
Finally, the **Audit Logger** captures asynchronous JSON telemetry across all pipeline stages, logging `LatencyMs`, `Verdict`, `Category`, `Confidence`, and `Cleaned Output` for centralized SIEM reporting.

## 📂 Project Structure
- **/go-backend:** Main Enterprise Firewall (Rate Limiter, Proxy Handlers, Audit Log, Output Filter).
- **/python-scrubber:** FastAPI Microservice for spaCy Dual-Pass NER PII Scrubbing.
- **/src:** React.js Administrator Dashboard & Attack Simulator lab to test threat injections live.

## 🚀 How To Run Locally

### 1. Start the React Frontend Dashboard
```bash
npm install
npm run dev
```
*(Runs on exactly `localhost:5173`)*

### 2. Start the Python PII Scrubber
```bash
cd python-scrubber
pip install -r requirements.txt # (fastapi, uvicorn, spacy, pydantic)
python -m spacy download en_core_web_sm
python -m uvicorn main:app --port 5001
```
*(Runs on exactly `localhost:5001`)*

### 3. Start the Go Firewall Proxy
Make sure you have an active Redis server (e.g., `redis-server` or via Docker on port `6379`).
Make sure Ollama is running (`llama3`) locally in the background on port `11434`.
```bash
cd go-backend
go run ./cmd/proxy/main.go
```
*(Proxy listens globally on `http://localhost:8080/v1/chat/completions`)*
